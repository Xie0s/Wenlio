# 状态管理与主题持久化说明

## 背景与目标

项目已将主题能力从函数式单例重构为 **Pinia Store**，并接入 `pinia-plugin-persistedstate`。目标是在不破坏既有调用方式的前提下，统一主题状态来源并稳定持久化行为。

## 规范项

### 1) 初始化顺序与入口接入

- 文件位置：`src/main.ts`
- 推荐顺序：
  1. `const pinia = createPinia()`
  2. `pinia.use(piniaPluginPersistedstate)`
  3. `initTheme(pinia)`
  4. `app.use(pinia)` 并挂载应用

> 说明：先初始化主题可保证首屏主题状态与 DOM 一致。

### 2) 主题 Store 设计（`src/stores/theme.ts`）

- 使用 `useThemeStore` 统一管理 `mode` 与 `resolvedTheme`。
- 在 Store 内集中处理：
  - 系统主题变化监听
  - `document.documentElement` 的 `dark` class 和 `data-theme` 同步
- 保留兼容函数 API（`initTheme / setTheme / toggleTheme / getResolvedTheme`），降低改造成本。

### 3) 持久化插件配置与迁移要点

- 插件版本：`pinia-plugin-persistedstate@4.x`
- 配置差异：
  - ✅ 新写法：`persist.pick`
  - ❌ 旧写法：`persist.paths`

当前主题持久化策略：

- `key: 'theme-store'`
- `pick: ['mode']`
- 同步保留 `localStorage.theme` 兼容字段，避免旧数据失效。

### 4) 组件侧使用规范

- `ThemeToggle`、`AnimatedThemeToggler`、`NoiseBackground` 统一直接消费 `useThemeStore`。
- 组件层只保留 UI 与交互。
- 状态、持久化、系统监听逻辑统一收敛在 `stores/theme.ts`。

## 示例（初始化流程）

1. 创建 Pinia 实例
2. 注册 persistedstate 插件
3. 调用 `initTheme(pinia)`
4. 挂载应用

## 维护建议

- 升级持久化插件时，优先核对配置字段是否仍为 `pick`。
- 主题相关新需求优先落到 `useThemeStore` 与 `stores/theme.ts`，避免回退到组件内分散实现。
