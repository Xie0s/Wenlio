# 布局稳定性与滚动锁定说明

## 背景与目标

为解决“弹出层打开导致滚动条消失，进而产生页面抖动（Layout Shift）”问题，采用分级策略，兼顾交互体验与布局稳定性。

## 规范项

### 1) 轻量交互组件：禁用滚动锁定

适用：`Select`、`Dropdown Menu`、`Context Menu` 等。

- `Dropdown Menu / Context Menu`：根组件显式配置 `modal: false`。
- `Select`：在 `SelectContent` 中显式配置 `bodyLock: false`。
- 结果：弹出层打开时背景滚动条保持不变，页面不抖动。

### 2) 模态交互组件：保留锁定 + 位移补偿

适用：`Dialog`、`AlertDialog` 等。

- 保持默认 `modal: true`。
- 组件库在锁定 body 滚动时自动计算滚动条宽度并注入 `padding-right` 补偿。
- 配合全局 6px 滚动条，进一步降低位移体感。

### 3) 全局配套策略

位于 `src/style.css`：

- 全局滚动条宽度统一为 `6px`。
- 滚动条轨道保持 `transparent`，减少锁定切换时的视觉空缺。

## 示例（策略映射）

- `Dropdown Menu / Context Menu`：`modal: false`
- `SelectContent`：`bodyLock: false`
- `Dialog / AlertDialog`：保留 `modal: true` + 自动补偿

## 维护建议

- 新增弹层组件时，先判断“轻量弹出”还是“模态交互”，再决定是否锁定滚动。
- 若出现布局偏移，优先排查是否误改滚动条宽度或覆盖了锁定补偿样式。
