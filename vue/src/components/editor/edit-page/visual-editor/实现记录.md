# 可视化编辑器设计方案

## 技术选型：Tiptap v2

选择 **Tiptap v2** 作为编辑器引擎，原因：
- **ProseMirror 内核**：工业级富文本编辑，稳定可靠
- **Vue 3 原生支持**：`@tiptap/vue-3` 提供无缝集成
- **Headless 设计**：完全自定义 UI，可复用现有 `prose` + [markdown.css](file:///e:/%E6%96%87%E6%A1%A3%E7%AE%A1%E7%90%86/doc-mather-1003/vue/src/assets/markdown.css:0:0-0:0) 样式
- **`tiptap-markdown`**：实现 Markdown ↔ ProseMirror 双向转换，保证保存格式与后端一致

## 数据流设计

```
页面选择 → editor.content (markdown string)
    ↓
preprocessContainers(): 将 :::type 语法转为 HTML
    ↓
tiptap.setContent(markdown) → ProseMirror 文档
    ↓ (用户所见即所得编辑)
tiptap.onUpdate → getMarkdown() → postprocessContainers()
    ↓
editor.content (原始 markdown string)
    ↓ (保存/自动保存)
API PUT/PATCH → MongoDB (content 字段，纯 markdown)
```

**关键保证**：编辑器始终以 **原始 Markdown** 与后端交互，前后端数据格式完全一致。

## 组件架构

```
visual-editor/
├── VisualEditorWorkspace.vue   # 主容器：工具栏 + 编辑区
├── VisualEditorToolbar.vue     # 富文本格式化工具栏（顶部）
├── VisualEditorContent.vue     # Tiptap 编辑器包装（prose 样式）

└── extensions/
    ├── index.ts                # 扩展配置集合
    └── CustomContainer.ts      # 自定义容器节点（:::tip/warning/danger/info）
```

## 工具栏设计（Word 风格，分组布局）

| 分组 | 功能 |
|------|------|
| 历史 | 撤销、重做 |

- 编辑区使用 `prose prose-slate dark:prose-invert max-w-none` 类 — **与现有预览/阅读完全一致**
- 导入 `@/assets/markdown.css` — **不做任何修改**
- Tiptap 的 `.ProseMirror` 编辑器继承 prose 容器的所有样式
- 自定义容器在编辑器中渲染与 `markdown-it-container` 输出相同的 HTML 结构
