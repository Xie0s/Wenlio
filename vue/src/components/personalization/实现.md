# 租户个性化首页 — 架构设计

## 一、需求梳理

### 核心目标
每个租户可自定义文档集合首页，由**可配置的区块（Section）** 组成，管理员在后台编辑保存，读者访问 `/:tenantId` 时看到个性化首页。

### 区块清单

| 区块 | 说明 | 关键能力 |
|------|------|----------|
| **Navbar** | 顶部导航 | 自定义 Logo/品牌名、导航链接（支持二级）、样式（透明/毛玻璃/纯色）、吸顶 |
| **Hero** | 首屏主视觉 | 多种布局（居中/左右分栏）、标题/副标题/描述、背景图+遮罩、CTA 按钮、入场动画 |
| **Introduction** | 特性介绍 | 图标+标题+描述的特性卡片、多列布局 |
| **ThemeList** | 文档主题列表 | 4 种卡片风格、列数可配、真实数据来源于 `store.themes` |
| **CTA** | 行动号召 | 标题/描述/按钮、背景色 |
| **Footer** | 页脚 | 品牌/版权、链接分组、社交媒体链接 |

### 数据流

```
管理员编辑 → PUT /tenant/homepage (保存草稿)
           → POST /tenant/homepage/publish (发布)

读者访问 /:tenantId → GET /public/tenants/:tenant_id/homepage
                     → HomepageRenderer 根据 config 渲染各区块
                     → ThemeList 区块自动从 store.themes 拉取真实数据
```

## 二、数据模型设计

### 为什么独立存储而非嵌入 Tenant

现有 `GET /public/tenants/:tenant_id` 被所有阅读页使用（面包屑、Logo 等），首页配置数据量大（JSON 可达数 KB），嵌入会使每次请求都携带无用负载。采用**独立端点**，仅首页加载时请求一次。

MongoDB 存储方案：在 `tenants` 文档中新增 `homepage` 字段（内嵌文档），但通过 **projection** 让现有接口不返回该字段，新接口只返回该字段。

### 后端新增结构

```go
// entity/tenant.go 扩展
type Tenant struct {
    // ... 现有字段 ...
    Homepage *TenantHomepage `bson:"homepage,omitempty" json:"-"` // json:"-" 现有接口不序列化
}

type TenantHomepage struct {
    Published *HomepageLayout `bson:"published" json:"published"`
    Draft     *HomepageLayout `bson:"draft"     json:"draft"`
    UpdatedAt time.Time       `bson:"updated_at" json:"updated_at"`
}

type HomepageLayout struct {
    Global   HomepageGlobal    `bson:"global"   json:"global"`
    Sections []HomepageSection `bson:"sections" json:"sections"`
}

type HomepageGlobal struct {
    BackgroundColor string `bson:"background_color" json:"background_color"`
    FontFamily      string `bson:"font_family"      json:"font_family"`
    MaxWidth        string `bson:"max_width"        json:"max_width"`
}

type HomepageSection struct {
    ID      string      `bson:"id"      json:"id"`
    Type    string      `bson:"type"    json:"type"`    // navbar|hero|introduction|theme_list|cta|footer
    Visible bool        `bson:"visible" json:"visible"`
    Config  interface{} `bson:"config"  json:"config"`  // BSON 存储为 raw document
}
```
