# 自定义滚动条规范

## 背景与目标

为解决 Windows 环境下原生滚动条视觉不协调问题，项目在 `src/style.css` 的 `base` 层进行了统一定制。

目标如下：

- 保持极简、低干扰。
- 保留 hover 反馈，增强可感知性。
- 在不同浏览器内核下保持一致体验。

## 规范项

### 1) 视觉规则

- 宽度固定为 `6px`。
- 轨道背景默认 `transparent`。
- 滑块静止状态使用 `--border` 颜色。
- 容器悬浮态（`*:hover`）滑块切换为 `--muted-foreground`。

### 2) 兼容实现

- Firefox：使用 `scrollbar-width` 与相关颜色属性。
- WebKit：使用 `::-webkit-scrollbar` 系列伪元素。

## 示例（规则映射）

- 结构尺寸：`6px`
- 默认状态：轨道透明 + 滑块 `--border`
- 交互状态：hover 时滑块 `--muted-foreground`

## 维护建议

- 调整滚动条视觉参数时，同步检查 Firefox 与 Chromium 浏览器，避免单内核生效。
- 如需改变宽度，优先评估对“布局稳定性与滚动锁定策略”的联动影响。
