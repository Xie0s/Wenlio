# 玻璃质感规范

## 背景与目标

项目在 `src/style.css` 中维护统一的玻璃质感工具类，以提升界面层级感，并减少组件各自实现造成的视觉不一致。

## 扁平化设计原则（必须遵守）

- 仅使用均匀半透明底色，不使用斜面高光、镜面反射或拟物层次。
- 不使用外部投影制造悬浮感，层次感由布局与留白表达。
- 玻璃感仅来源于 `backdrop-filter` 对底层内容的采样与模糊。
- 禁止新增视觉强弱变体，统一只保留 `.glass`。

## 规范项

### 1) 工具类用途

| 类名 | 适用场景 | 模糊强度 |
|------|---------|---------|
| `.glass` | 导航栏、工具条、信息卡片、弹窗、Tooltip 等全部玻璃场景 | `blur(16px)` |

### 2) 实现要点

- 使用 `backdrop-filter: blur()` 构建模糊层，结合半透明 `oklch` 背景色控制通透感。
- **`-webkit-backdrop-filter` 必须写在 `backdrop-filter` 之前**（见下方坑点说明）。
- 所有 glass 类的 `border` 均为白色/主色半透明，亮色模式下几乎不可见，
  需要可见边框时应叠加 `ring-inset`（见下方边框规范）。

### 3) `-webkit-` 属性顺序要求（⚠️ 重要）

**现象**：`npm run build` 后毛玻璃变为纯透明，开发环境正常。

**根因**：Vite 生产构建使用 Lightning CSS 优化 CSS。当 `backdrop-filter`
写在 `-webkit-backdrop-filter` **之前**时，Lightning CSS 会将标准属性优化掉，
仅保留 `-webkit-` 版本，导致生产环境失去毛玻璃效果。

**正确顺序**（`-webkit-` 降级在前，标准属性在后）：

```css
/* ✅ 正确 */
.glass {
  -webkit-backdrop-filter: blur(16px) saturate(160%);
  backdrop-filter: blur(16px) saturate(160%);
}

/* ❌ 错误 — 生产构建会丢失 backdrop-filter */
.glass {
  backdrop-filter: blur(16px) saturate(160%);
  -webkit-backdrop-filter: blur(16px) saturate(160%);
}
```

> 已于 2026-02 统一修复 `src/style.css` 的 `.glass` 属性顺序。

### 4) 搭配 Tailwind 的边框规范

`.glass` 工具类的 `border` 使用 CSS **简写属性**，会覆盖后续通过 Tailwind
`border-color` 类添加的颜色。如需在亮色/暗色模式下显示可见边框，
**应使用 `ring-inset` 而非 `border-*`**：

```html
<!-- ✅ 正确：ring-inset 基于 box-shadow，与 border 属性正交 -->
<div class="glass ring-[0.5px] ring-inset ring-black/[0.07] dark:ring-white/[0.08]">

<!-- ❌ 错误：border-color 会被 glass 的 border 简写覆盖 -->
<div class="glass border-black/10">
```

## 示例（使用建议）

- 普通信息卡片：使用 `.glass`
- 模态或提示层：使用 `.glass`
- 顶部导航背景：使用 `.glass`
- 主操作强调区：使用 `.glass`（如需品牌强调，叠加 `ring` 或文本色，不新增 glass 变体）
- 鼠标跟随 Tooltip：`.glass` + `ring-[0.5px] ring-inset`（参见 [TooltipCard 使用规范](./TooltipCard使用规范.md)）

## 禁用场景

- 禁止在 glass 元素上叠加不透明的 `bg-*` 类（会破坏半透明背景）。
- 禁止在页面层直接书写 `backdrop-filter` 内联样式，统一走工具类。
- 禁止在 glass 元素的祖先容器上设置 `isolation: isolate`（会阻断 backdrop 渗透）。

## 维护建议

- 禁止新增 `glass-*` 变体，统一仅使用 `.glass`。
- `.glass` 必须保持扁平化（无斜面高光、无外投影），玻璃感仅由 `backdrop-filter` 提供。
- 页面层仅消费已有工具类，不直接写散落的玻璃样式实现。
