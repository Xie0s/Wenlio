## 1. 文件命名规范

- 统一使用**中文命名**。
- 文件名建议使用“主题 + 类型”格式：
  - `xx规范.md`
  - `xx说明.md`
  - `xx模板.md`
- 不使用英文缩写命名新文档（如 `xxx-guide.md`、`spec.md`）。

## 2. 文档头部注释格式（必填）

每个新文档都必须包含如下头部注释：

```md
<!--
文档：xxx
职责：说明该文档的职责与边界（负责什么，不负责什么）。
主要接口：
  - 列出该文档提供的关键内容（如：规范项、模板、检查清单）
-->
```

## 3. 正文结构建议

推荐结构（按需增减）：

1. `# 标题`
2. `## 背景/目标`
3. `## 规范项/实现说明`
4. `## 示例`
5. `## 维护建议`

> 要求：内容清晰、可执行、避免口语化描述。

## 4. 新增文档时必须更新的内容

新增任意文档后，至少完成以下同步：

1. 更新 `index.md` 的“目录大纲（唯一入口）”列表：
   - 增加新文档入口链接
   - 补充一句简要说明
2. 检查是否与现有文档重复：
   - 若重复，优先合并内容
   - 避免同主题多文件长期并存
3. 若涉及重命名：
   - 旧文件改为“文档已迁移”提示
   - 指向新文件，避免历史链接失效

## 5. 变更自检清单

提交前建议检查：

- [ ] 文件名是否为中文、语义明确
- [ ] 是否包含头部注释（文档/职责/主要接口）
- [ ] `index.md` 的目录大纲是否已补充入口
- [ ] 是否存在重复文档或重复段落
- [ ] 链接是否可点击跳转

## 6. 维护边界

- `index.md`：作为唯一目录索引与维护规范入口。
- `样式规范说明.md`：仅保留迁移提示，不再维护目录内容。
- 其他专题文档：承载实际规范内容。
- 本文档：只维护“如何写文档、更新哪些文件”的流程规范。
