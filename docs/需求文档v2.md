# 文档管理平台 产品需求文档

> 版本：v2.0  
> 日期：2026-02-27  
> 变更说明：放弃 VitePress 静态构建方案，改为 Vue 3 单应用（动态渲染 Markdown），极简架构

---

## 一、产品概述

### 1.1 产品定位

一个支持多租户的在线文档管理与阅读平台。租户管理员通过 Web 界面创建和管理文档，读者通过同一应用的阅读界面浏览文档——**无需任何构建步骤，发布即生效**。

### 1.2 核心特性

- **单应用架构**：管理界面与文档阅读界面同属一个 Vue 3 应用，Go 后端统一服务
- **即时发布**：发布操作仅更改文档状态，无构建等待，读者刷新即可看到最新内容
- **多租户隔离**：每个租户拥有独立的主题、文档、版本数据，路径和数据完全隔离
- **Markdown 渲染**：前端动态渲染 Markdown，支持代码高亮、自定义容器、表格等完整语法
- **版本管理**：支持同一主题维护多个文档版本，读者可切换版本阅读
- **全文搜索**：基于 Go 后端的全文搜索，按租户隔离结果
- **文档评论**：读者可在文档页发表评论，管理员审核管理

### 1.3 访问地址结构

```
https://microswift.cn/admin/                                    ← 管理后台（登录后可用）
https://microswift.cn/{tenant_id}/                             ← 租户文档首页（主题列表）
https://microswift.cn/{tenant_id}/{theme_slug}/                ← 主题首页（默认版本）
https://microswift.cn/{tenant_id}/{theme_slug}/{version}/      ← 指定版本首页
https://microswift.cn/{tenant_id}/{theme_slug}/{version}/{page_slug}  ← 具体文档页
```

## 三、多租户设计

### 3.1 隔离策略

| 隔离维度     | 方案                                                 |
| ------------ | ---------------------------------------------------- |
| **路径隔离** | 所有文档 URL 以 `/{tenant_id}/` 为前缀               |
| **数据隔离** | 所有集合携带 `tenant_id` 字段，Repository 层强制过滤 |
| **权限隔离** | JWT Token 携带 `tenant_id`，接口层校验不得跨租户访问 |
| **搜索隔离** | 搜索接口强制限定 `tenant_id` 条件                    |

### 3.2 租户数据结构

```
Tenant {
  id          string    // URL 路径段，如 "acme"
  name        string    // 显示名称
  logo_url    string
  status      enum      // active / suspended
  created_at  datetime
}
```

> `id` 即 URL 中的 `{tenant_id}`，创建后不可修改。

## 四、文档结构

### 4.1 文档层级

```
租户（Tenant）
  └── 主题（Theme）         ← 一个产品或文档分类
        └── 版本（Version） ← 如 v1.0、v2.0、latest
              └── 章节（Section）  ← 侧边栏分组
                    └── 文档页（Page）
```

### 4.2 数据结构

**Theme（主题）**

```
Theme {
  id          string
  tenant_id   string
  name        string
  slug        string    // URL 段，如 "api-reference"
  description string
  icon        string
  sort_order  int
  created_at  datetime
}
```

**Version（版本）**

```
Version {
  id           string
  tenant_id    string
  theme_id     string
  name         string    // 如 "v1.0"、"latest"
  label        string    // 展示标签，如 "1.0（稳定版）"
  status       enum      // draft / published / archived
  is_default   bool      // 访问主题时默认展示此版本
  created_at   datetime
  published_at datetime
}
```

**Section（章节）**

```
Section {
  id          string
  tenant_id   string
  version_id  string
  title       string
  sort_order  int
}
```

**Page（文档页）**

```
Page {
  id          string
  tenant_id   string
  version_id  string
  section_id  string
  title       string
  slug        string    // URL 段，如 "quick-start"
  content     string    // Markdown 原文，存储在 MongoDB
  status      enum      // draft / published
  sort_order  int
  created_at  datetime
  updated_at  datetime
  published_at datetime
}
```

## 六、文档编辑器

### 6.1 编辑器功能

- **双模式**：富文本（WYSIWYG）+ Markdown 源码模式，随时切换，内容互转
- **内容支持**：
  - 标题（H1～H4）、段落、粗体、斜体、删除线
  - 有序/无序列表、任务列表
  - 代码块（含语言选择，语法高亮预览）
  - 行内代码、引用块、表格、分隔线、外链
  - 图片（拖拽/粘贴上传至 Go 文件接口）
  - 自定义容器（Tip / Warning / Danger / Info）
- **导入 Markdown**：上传本地 `.md` 文件，自动解析 frontmatter 中的 `title`、`description`
- **自动保存**：每 30 秒自动保存草稿到后端
- **实时预览**：编辑区右侧显示渲染效果（与阅读界面渲染一致）

## 七、文档阅读界面（Vue 3 渲染）

### 7.1 页面布局

```
┌────────────────────────────────────────────────────┐
│  顶部导航栏                                          │
│  [主题名称]  [版本切换下拉]  [搜索框]               │
├──────────────┬─────────────────────────────────────┤
│              │                                     │
│  侧边栏      │  文档正文                            │
│  （章节树）  │  Markdown 渲染内容                  │
│              │                                     │
│              ├─────────────────────────────────────┤
│              │  评论区                              │
└──────────────┴─────────────────────────────────────┘
```

### 7.2 Markdown 渲染能力

前端使用 `markdown-it` + `shiki` 实现与 VitePress 一致的渲染效果：

| 功能              | 实现方式                                           | 说明                                                          |
| ----------------- | -------------------------------------------------- | ------------------------------------------------------------- |
| Markdown 解析     | `markdown-it`（`html: true`）                      | 允许嵌入 HTML，与 VitePress 行为一致                          |
| XSS 防护          | `DOMPurify`                                        | 对渲染输出 HTML 进行净化，过滤危险内容                        |
| Markdown 排版样式 | `@tailwindcss/typography`（`prose` 类）            | 恢复 TailwindCSS Reset 清零的标题/列表/表格样式，**必须引入** |
| 代码语法高亮      | `shiki`（`@shikijs/markdown-it`，200+ 语言）       | 双主题（github-light / github-dark），CSS 变量切换            |
| 自定义容器        | `markdown-it-container`（Tip/Warning/Danger/Info） | 需配套 `markdown.css` 定义容器样式                            |
| 标题锚点          | `markdown-it-anchor`                               | 与 TOC 配合使用                                               |
| 目录生成          | `markdown-it-toc-done-right`（右侧浮动目录）       | `[[toc]]` 占位符自动替换                                      |
| 表格              | markdown-it 原生支持                               | —                                                             |
| 数学公式          | `markdown-it-mathjax3`（选配）                     | 按需引入                                                      |
| 暗色模式          | TailwindCSS `dark:prose-invert` + Shiki CSS 变量   | `<html class="dark">` 控制，无需重渲染                        |

**自定义容器渲染示例：**

```md
::: tip 提示
这是一个提示框
:::

::: warning 注意
这是警告内容
:::
```

### 7.3 数据加载方式

- 首次访问文档页：Vue 3 调用 Go API 获取页面内容（Markdown 原文）
- 前端渲染：`markdown-it` 将 Markdown 转为 HTML，注入页面
- 侧边栏：调用 API 获取当前版本的章节/文档树
- **客户端缓存**：相同页面内容缓存在 Pinia store，切换页面无重复请求

### 7.4 版本切换

- 顶部导航栏版本下拉：列出该主题所有 `published` + `archived` 版本
- 切换版本时跳转至相同 `slug` 的页面（不存在则跳版本首页）
- 版本切换**不刷新整页**，通过 vue-router 导航更新内容区域

## 八、版本控制

### 8.1 版本操作

| 操作         | 说明                                           |
| ------------ | ---------------------------------------------- |
| **创建版本** | 基于现有版本克隆全部章节和文档页（深拷贝）     |
| **编辑版本** | `draft` / `published` 状态的版本可编辑         |
| **发布版本** | 状态改为 `published`，读者立即可见，无需构建   |
| **设为默认** | 访问 `/{tenant_id}/{theme_slug}/` 时展示此版本 |
| **归档版本** | 状态改为 `archived`，只读，读者仍可访问        |

### 8.2 发布粒度

- **单页发布**：单独发布/下线某一篇文档，不影响其他页面
- **版本发布**：将整个版本的所有 `draft` 文档批量发布

## 九、全文搜索

### 9.1 方案

利用 Go 后端的 **MongoDB `$text` 索引**实现全文搜索，无需引入额外搜索中间件。

- `pages` 集合在 `title` + `content` 字段建立文本索引（标题权重 10，内容权重 1）
- 搜索接口**无需 JWT 鉴权**，读者可直接使用，归入 `/public/` 路由组
- 强制携带 `tenant_id` 查询参数，服务端校验存在性，天然隔离租户数据
- 支持按主题、版本过滤

**搜索 API：**

```
GET /api/v1/public/search?q={keyword}&tenant_id={tenant_id}&theme_id={可选}&version_id={可选}
```

返回：标题、匹配片段（高亮关键词）、文档路径（`/{tenant_id}/{theme_slug}/{version}/{page_slug}`）

### 9.2 搜索 UI

- 文档阅读界面顶部导航栏集成搜索框
- 输入关键词后展示结果弹窗（实时搜索，防抖 300ms）
- 结果展示：文档标题、所属主题/版本、内容摘要（高亮匹配词）

## 十、文档评论

### 10.1 评论数据结构

```
Comment {
  id          string
  tenant_id   string
  page_id     string
  parent_id   string       // 回复时指向父评论，顶层为空
  author {
    name      string
    email     string       // 选填，不公开展示
  }
  content     string       // 纯文本，限 1000 字
  status      enum         // pending / approved / rejected
  created_at  datetime
}
```

### 10.2 评论流程

1. 读者在文档页底部填写昵称（必填）、邮箱（选填）、内容，点击提交
2. 状态默认 `pending`，**不公开展示**，等待管理员审核
3. 管理员在后台"评论管理"审核：批准 → `approved`（公开），拒绝 → `rejected`
4. 支持一层嵌套回复（管理员回复直接公开，无需审核）

### 10.3 评论展示

- 文档页末尾展示 `approved` 状态评论列表（Vue 3 组件，调用 Go 公开 API）
- 展示：楼层顺序、昵称、时间、内容、回复

## 十一、API 接口概要

### 11.1 认证

- **JWT** 认证，Token 携带 `user_id`、`role`、`tenant_id`
- 所有 `/api/v1/admin/*` 和 `/api/v1/tenant/*` 接口需鉴权
- `/api/v1/public/*` 无需鉴权（已发布内容）

### 11.2 接口分组

**鉴权接口（需 JWT Token）：**

| 路径                                  | 说明                       |
| ------------------------------------- | -------------------------- |
| `/api/v1/auth/*`                      | 登录、刷新 Token、修改密码 |
| `/api/v1/admin/tenants`               | 超级管理员：租户 CRUD      |
| `/api/v1/admin/users`                 | 超级管理员：账号管理       |
| `/api/v1/tenant/themes`               | 租户管理员：主题管理       |
| `/api/v1/tenant/versions`             | 租户管理员：版本管理       |
| `/api/v1/tenant/sections`             | 租户管理员：章节管理       |
| `/api/v1/tenant/pages`                | 租户管理员：文档页 CRUD    |
| `/api/v1/tenant/pages/import`         | 导入 Markdown 文件         |
| `/api/v1/tenant/pages/:id/publish`    | 发布/下线单页              |
| `/api/v1/tenant/versions/:id/publish` | 批量发布版本               |
| `/api/v1/tenant/comments`             | 评论审核管理               |
| `/api/v1/tenant/media/upload`         | 图片/媒体文件上传          |

**公开接口（无需鉴权，`/public/` 路由组）：**

| 路径                                           | 说明                                                   |
| ---------------------------------------------- | ------------------------------------------------------ |
| GET `/api/v1/public/tenants/:tenant_id`        | 租户基础信息（名称、Logo）                             |
| GET `/api/v1/public/tenants/:tenant_id/themes` | 租户已发布主题列表（读者首页）                         |
| GET `/api/v1/public/themes/:theme_id/versions` | 主题的 published/archived 版本列表                     |
| GET `/api/v1/public/versions/:version_id/tree` | 版本章节+文档树（侧边栏，含 slug/title，不含 content） |
| GET `/api/v1/public/pages/:page_id`            | 单页 Markdown 内容（仅 published）                     |
| GET `/api/v1/public/pages/:page_id/comments`   | 页面 approved 评论列表                                 |
| POST `/api/v1/public/pages/:page_id/comments`  | 提交评论（默认 pending）                               |
| GET `/api/v1/public/search`                    | 全文搜索（必须携带 `tenant_id` 参数）                  |
