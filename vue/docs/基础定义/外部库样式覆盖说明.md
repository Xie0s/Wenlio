# 外部库样式覆盖说明

## 背景与目标

第三方 UI 库默认样式通常与项目视觉系统不一致。本规范用于沉淀统一覆盖策略，确保升级后仍可快速回归。

## 规范项

### 1) 覆盖对象与入口

- 覆盖对象：Sonner（Toast）
- 文件位置：`src/style.css`（底部"Sonner Toast 毛玻璃样式"区块）
- 覆盖目标：让 Toast 与项目整体视觉语言一致。

### 2) 覆盖规则

- 应用与 `.glass` 工具类一致的毛玻璃质感（`blur(16px) saturate(180%)`）。
- 移除原生默认 `box-shadow`。
- 使用 `border` 轮廓线替代阴影（亮色黑色调 / 暗色白色调）。
- 圆角统一为 `1rem`（`rounded-2xl`）。
- 选择器前缀使用 `:root` / `:root.dark` 提升特异性，覆盖 vue-sonner 运行时注入样式。

## 示例（覆盖点清单）

- 质感：`.glass`（`oklch(1 0 0 / 60%)` + backdrop-filter）
- 阴影：移除（`box-shadow: none`）
- 描边：`border: 1px solid`（亮色 `oklch(0 0 0 / 8%)`，暗色 `oklch(1 0 0 / 8%)`）
- 圆角：`1rem`

> 详细 API 用法与使用约束见 [Sonner通知使用规范](./Sonner通知使用规范.md)。

## 维护建议

- 升级 `vue-sonner` 版本后，优先回归验证毛玻璃效果与圆角是否仍正常显示。
- 避免在页面级组件重复覆盖 Toast 样式，统一在 `src/style.css` 专属区块维护。
